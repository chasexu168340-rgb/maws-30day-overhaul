版本：Story Integration Devbook v1.0  
目标仓库：`chasexu168340-rgb/maws-30day-overhaul`

## 1. 当前项目判断

当前项目已经不是空白原型，而是有一套能运行的 30 天武术实用性 Web demo。现有结构很适合接入主线，不应该重写。

当前事实：

- 架构是 `Phaser + DOM`：Phaser 负责背景、角色 sprite、动画、伤害数字和战斗特效；DOM 负责主线/待办、装备栏、训练小游戏、战斗 HUD、技能卡、队列、日志和弹窗。
- 入口是 `maws_30day_overhaul_v3.html`。
- 数据集中在 `maws_src/content/data.js`。
- 状态在 `maws_src/simulation/state.js`。
- 战斗在 `maws_src/simulation/combat.js`。
- DOM UI 在 `maws_src/dom/ui.js` 与 `maws_src/dom/ui.css`。
- 机会卡在 `maws_src/simulation/events.js`。
- 项目已经有 30 天框架、地点、出行、训练、商店、装备、NPC、战斗、主线事件、机会卡、结算弹窗和移动端适配。

所以主线接入原则是：

> 不新增庞大架构，不推翻现有 DOM + Phaser 分层；通过数据、状态字段、主线日程、拳谱 UI、战斗目标和文案，把项目从“30 天武术实用性原型”升级成“P5 式日历叙事 RPG 原型”。

## 2. 主线核心重新定义

最终主线一句话：

> 主角从小被父亲用“茂家拳天下第一”的善意谎言养大，靠体型、运气和几次误判胜利相信自己是传统武术天才；在真正对抗中被“一阵风”打醒后，他发现茂家拳原本是假的，却决定从自己这一代开始，把它练成真的。

这个核心要同时服务三个层面：

1. **叙事层**：强人设、误判、自尊崩塌、父亲日记、百家入茂、最终承认“我就是茂拳”。
2. **玩法层**：30 天倒计时，玩家自由安排训练、赚钱、恢复、社交、战斗。
3. **系统层**：所有成长都回流到“茂家拳谱重铸”，而不是学了一堆散技能。

---

## 3. Demo 的最终玩家体验

玩家开局不是“普通菜鸟”，而是一个看起来很有气势、自己也坚信很强的人。

玩家前 7 天会经历几次“误判胜利”。系统表层给玩家自信、名声、士气，底层积累“误判值”。  
第 8 天发生第一次真实对抗，玩家被打出“一阵风”。  
第 9 天翻到父亲日记，知道茂家拳不是神功，而是父亲为了让小时候胖、自卑的自己站起来而编出的善意谎言。  
第 10 到 28 天进入自由训练期。玩家通过拳馆、武馆、MMA 馆、公园、便利店、工地、理疗店等地点，把拳击、传统拆解、地面防摔、街头判断逐步写回茂家拳谱。  
第 29 天烧烤摊夜谈。  
第 30 天真东西测试。最终目标不是 KO，而是证明自己不再是一阵风就倒的人。

Demo 胜利定义：

> 你没有证明茂家拳天下第一，但你证明了它终于不是零。

## 4. 现有系统接入点

### 4.1 `data.js`

需要接入：

- 新增 `MAW_RULES`：解释茂家拳相关资源。
- 新增 `MAW_FORMS`：拳谱旧解/新注。
- 替换或扩展 `MAIN_EVENTS`：从当前 Day 1 / 5 / 12 / 18 / 24 / 30，升级到更完整的 Day 1 / 2 / 3 / 5 / 8 / 9 / 12 / 18 / 24 / 29 / 30。
- 扩展 `NPCS`：增加父亲、阿豹、推手大爷等叙事节点；父亲可不进入关系页，只进入拳谱/日记。
- 扩展 `ITEMS`：加入父亲旧护腕、祖传拳谱、直播间秘传护腕、正规缠手、护膝等。
- 扩展 `ENEMIES`：增加 Day 8 “一阵风”专用对手，或复用 E01 但加 `script: 'first_wind'`。
- 扩展 `ACTIONS`：出租屋新增“给父亲上香”“翻看拳谱”；地铁或旧城区新增误判胜利；夜市/烧烤摊新增 Day 29 夜谈。

### 4.2 `state.js`

需要接入：

- `createNewState()` 新增 `player.maw` 或 `maw` 根字段。
- `migrateSave()` 给旧存档补字段。
- `gainParts()`、`settlementLines()` 支持新增资源的显示。
- `startMainEvent()` 支持剧情特殊类型，例如 `scripted`, `choice`, `diary`, `nightTalk`。
- `startBattle()` 支持 `script`、`objectives`、`forcedOutcome`。
- `finishBattle()` 支持目标制胜利，不只看 HP。
- `postReview()` 可以把战斗复盘转化为拳谱新注。
- `buildRenderModel()` 输出 `maw`、`deadline`、`chapterTitle`、`punchbook` 等 UI 所需字段。

### 4.3 `events.js`

需要接入：

- 机会卡改为服务主线状态。
- 误判期机会卡偏搞笑和误导。
- 一阵风后机会卡偏训练、复盘、父亲记忆、NPC 陪伴。
- 热度高时刷挑战。
- 真实性低时刷玄学诱惑。
- 疲劳/伤病高时刷理疗提醒。
- 父亲记忆高时刷日记边角内容。

### 4.4 `combat.js`

需要接入：

- Day 8 战斗的“剧情性失败”。
- Day 30 战斗的“目标制验收”。
- 技能重铸对风险、命中、冷静、目标判定的影响。
- 战斗 step 中加入更多叙事标签，例如 `firstWind`, `reforgeHit`, `fatherMemoryProc`, `objectiveComplete`。
- 注意：当前 `combat.js` 内部也有 frozen `SKILLS` 和 `ENEMY_TEMPLATES`。如果只改 `data.js`，战斗预览和实际结算可能不一致。应当两边同步，或在后续 Batch 做一次小型统一引用。

## 5. 新增核心状态字段

建议新增根字段 `maw`，不要把所有内容塞进 `player`，便于以后做拳谱 UI。

```js
maw: {
  chapter: 'illusion',
  belief: 75,
  misread: 0,
  fatherMemory: 0,
  reforge: 0,
  truthRevealed: false,
  firstWindDone: false,
  diaryRead: false,
  forms: {
    openMountain: { old: true, reforge: 0, unlockedNote: false },
    ironBody: { old: true, reforge: 0, unlockedNote: false },
    sealThroat: { old: true, reforge: 0, unlockedNote: false },
    windStep: { old: true, reforge: 0, unlockedNote: false }
  },
  modules: {
    boxing: 0,
    body: 0,
    traditional: 0,
    grappling: 0,
    street: 0
  },
  objectives: {}
}
```

字段含义：

- `belief`：祖传信念。前期高。Day 9 后不再是盲信，转化为父亲记忆与不服输。
- `misread`：误判值。前期靠误判胜利上涨；Day 8 会放大摧毁感。
- `fatherMemory`：父亲记忆。解锁日记、文案、烧烤夜谈、最终冷静加成。
- `reforge`：茂拳完成度。由各模块综合计算，不建议直接刷。
- `forms`：拳谱旧解/新注。
- `modules`：百家入茂模块，来自拳击、身体、传统拆解、地面防摔、街头判断。
- `objectives`：Day 30 终战目标记录。

## 6. 30 天主线重排

现有 `MAIN_EVENTS` 可以替换为下面的节奏。为了不扩大项目范围，每天仍只保留一个主线节点，自由行动通过 `EVENT_RULES` 补充。

| 天数 | 标题 | 地点 | 类型 | 目的 |
|---|---|---|---|---|
| 1 | 给父亲上香 | home | diary/dialog | 建立父亲、祖训、茂家拳人设 |
| 2 | 地铁见义勇为 | street 或 home 弹窗 | scripted/choice | 第一次误判胜利 |
| 3 | 便利店货架事件 | store | choice | 220 斤威慑成功，误判+1 |
| 5 | 公园第一次验货 | park | battle | 看似低风险胜利/输赢都误判 |
| 8 | 一阵风 | boxing | battle/scriptedLoss | 真正现实打脸 |
| 9 | 父亲日记 | home | diary | 揭露善意谎言，开启百家入茂 |
| 12 | 阿豹刷到你了 | park/street | battle/dialog | 短视频信徒镜像 |

## 7. UI 接入方案

### 7.1 HUD

当前 HUD 显示日期、阶段、地点、资源。建议新增一个红色 deadline chip：

```text
第12天 周五 18:40
百家入茂 · 夜晚 · 拳馆
距真东西测试还有18天
```

资源行保留，但把“真实性”“热度”“体能沉淀”与新 `maw` 资源分层：

- 主 HUD：现金、体力、健康、士气、真实性、热度、距终战。
- 人物页/拳谱页：祖传信念、误判值、父亲记忆、茂拳完成度。

原因：HUD 太满会毁掉当前移动端布局。

### 7.2 地图页任务板

地图页右侧任务板改成三段：

1. 今日主线：不可忽略，有地点与倒计时。
2. 今日自由时间：显示今天还适合做什么，例如训练/恢复/赚钱/社交。
3. 机会卡：由 `events.js` 生成。

示例：

```text
今日主线
一阵风
地点：拳馆
你要去证明茂家拳。系统建议：这不是一个好主意。
[前往拳馆]

今日自由时间
疲劳 64，建议先恢复；但主线不会等你变完美。

机会卡
- 梁教练暂停你的玄学发挥
- 网购页面开始对你眨眼
- 身体向你递交投诉书
```

### 7.3 新增“茂家拳谱”区块

不一定新增 tab。建议放在人物页顶部或技能页上方，初期做成卡片即可。

拳谱卡结构：

```text
茂家开山拳
旧解：一拳开山，气吞山河。
状态：旧解可用，但高风险。
新注：先别开山，先把手收回来护脸。
重铸来源：拳馆 · 直拳 / 抱架 / 回收
重铸进度：42%
```

Day 9 前只显示旧解；Day 9 后出现新注系统。

### 7.4 Day 8 “一阵风”表现

UI 表现要简单但强：

1. 战斗开场正常。
2. 敌人第一记有效刺拳命中。
3. Phaser 做白闪 + 角色后仰。
4. DOM 战斗记录短暂停顿。
5. 弹窗出现：

```text
一阵风

你看见他的肩膀动了一下。
然后世界安静了。

爸，我只感觉到一阵风。
```

按钮：

- `醒过来`
- `嘴硬一下`（只改文案，不改变失败）

### 7.5 Day 9 父亲日记

父亲日记不要做普通 dialog，要做沉浸式 modal：

```text
父亲日记

儿子，如果你看到这里，大概是终于遇到真正会打的人了。
爸不是不教你绝招。
爸是真的不会。

茂家拳不是天下第一。
但你第一次愿意出门跑步，是因为你相信它。
你第一次没有低着头走过校门，也是因为你相信它。

如果真的遇到高手，退。
不是丢人，是回家。
```

最后出现按钮：

- `原来都是假的`
- `至少他是真的希望我站起来`
- `从我开始，把它练成真的`

第三个按钮出现后进入第二幕。

## 8. 战斗接入方案

### 8.1 Day 8 “剧情性失败”

当前战斗系统已经支持逐招交换和 `finishBattle()`。不要硬改成完全 cutscene。建议做：

- `MAIN_EVENTS[8]` 写 `script: 'first_wind'`。
- `startBattle(state, enemyId, main)` 读取 mainEvent 的 script，写入 `state.combat.script`。
- `resolveCombatExchange()` 正常打一窗口。
- `mergeCombatResult()` 后，如果 `combat.script === 'first_wind'` 且窗口数 >= 1，则强制 `finishBattle(state, 'first_wind')`。
- `finishBattle()` 对 `first_wind` 走专用结果：不扣到濒死，不给普通失败弹窗，不给奖励，给父亲日记前置 flag。

### 8.2 Day 30 目标制终战

目标制比 KO 更适合 Demo。

目标建议 6 个，完成 3 个算“阶段胜利”，完成 5 个算“漂亮通过”。

```js
objectives: {
  surviveWindow1: false,
  guardHeavy: false,
  landStraight: false,
  recoverFromHit: false,
  useReforgedSkill: false,
  keepCalm: false
}
```

目标触发：

- `surviveWindow1`：完成第一个自动窗口且 HP > 0。
- `guardHeavy`：敌方重击时玩家使用 `guard/sprawl/dodge` 有效。
- `landStraight`：`straight` 或重铸后的“开山拳新注”命中。
- `recoverFromHit`：被击中后下一窗口仍完成有效动作。
- `useReforgedSkill`：使用拳谱重铸进度 >= 60 的技能。
- `keepCalm`：战斗结束时 `calm >= 40`。

最终结算文案：

- 0-1 个目标：你还是被现实打散了，但这一次你知道散在哪里。
- 2 个目标：你输了，但不是一阵风。
- 3-4 个目标：你通过了真东西测试。
- 5-6 个目标：茂家拳第一次配得上自己的名字。

## 9. 数值接入方案

现有数值不要大改。新增数值应依附现有系统。

### 9.1 误判值 `misread`

获得来源：

- Day 2 地铁误判胜利：+12
- Day 3 便利店威慑：+8
- 打赢低风险但未复盘：+4
- 买玄学装备：+6
- 评论区嘴硬：+5

影响：

- Day 8 惨败文案更刺痛。
- 高误判降低 Day 8 前 `auth`。
- Day 9 后误判值清零或转化为 `fatherMemory` 的一部分。

建议公式：

```js
const converted = Math.round(state.maw.misread * 0.35);
state.maw.fatherMemory += converted;
state.maw.misread = 0;
```

### 9.2 父亲记忆 `fatherMemory`

获得来源：

- 给父亲上香：+3
- 阅读日记：+25
- 使用父亲旧护腕：+2
- 烧烤摊独处：+8
- 终战前选择真诚路线：+5

影响：

- 提高最终战 `keepCalm` 判定。
- 解锁拳谱新注文案。
- 某些对话多一个温柔选项。
- Day 30 如果父亲记忆高，失败结尾也不显得崩。

### 9.3 茂拳完成度 `reforge`

不直接刷，按模块计算：

```js
reforge = weightedAverage([
  modules.boxing,
  modules.body,
  modules.traditional,
  modules.grappling,
  modules.street
])
```

MVP 权重：

- 拳击 30%
- 身体控制 20%
- 传统拆解 20%
- 地面防摔 20%
- 街头判断 10%

这样 Demo 不会鼓励只刷一处。

## 10. 装备接入方案

装备要分三类：

### 10.1 正规装备

继续保持现实感。

- 入门拳套
- 防滑训练鞋
- 护齿
- 训练笔记本
- 正规缠手
- 护膝

作用：实际影响战斗或恢复。

### 10.2 情感装备

不强，但有叙事价值。

- 父亲旧护腕：`fatherMemory + 1`，`calm + 2`，装备时终战失败文案不同。
- 祖传拳谱：不可消耗，开启拳谱页。
- 父亲跌打酒：轻恢复 + 父亲记忆。

### 10.3 玄学装备

好笑但有代价。

- 直播间秘传护腕：士气 +8，误判 +6，真实性 -2。
- 开光护膝：心理安慰 +5，膝盖保护 +1。
- 祖传内劲贴：疲劳 -3，热度 +1，真实性 -1。

玄学装备不是纯废物，它给心理收益，但不应该提升核心战斗能力。这会和主线“假的也可能陪你站起来，但不能替你打”一致。

## 11. NPC 接入方案

### 刘胖子

功能：朋友、吐槽、流量线入口。  
前期强化主角强人设，中后期帮他接受真相。  
典型台词：

```text
你这个气势，确实像高手。
就是高手一般不会把便利店货架拍得道歉。
```

### 小满

功能：现实生活锚点、便利店事件、非暴力处理。  
她提醒玩家：不是所有冲突都该打。

```text
你真要帮我，就别把门口打成决赛现场。
监控开着呢，警察也不是观众席。
```

### 梁教练

功能：拳击基础、现实检验、毒舌老师。  
负责“开山拳新注”。

```text
你这不叫拳，叫用肩膀发表意见。
先把手收回来。脸不是公共资源。
```

### 周青山

功能：传统拆解，不是骗子，也不神化。  
负责“封喉手新注”和“传统不是不能用，是不能只剩名字”。

```text
招名可以留，神话先放一边。
落到压力里还剩下的，才叫东西。
```

### 推手大爷

功能：身体控制、平衡、幽默。  
负责“铁布衫新注”。

```text
你这不叫下盘稳。
你这是上半身迷路了，下半身不敢报警。
```

### 阿豹

功能：短视频信徒，主角镜像。  
他不是坏人，是另一个被视频喂大的年轻人。  
可作为 Day 12 对手或 Day 30 前置挑战者。

## 12. 文案风格规则

文案基调：

> 诙谐，有梗，但不油；敢损主角，但不羞辱主角；笑点之后必须有情感落点。

写法规则：

1. 描写动作时，先给画面，再给损友式旁白。
2. 失败不要只写失败，要写“失败提供了什么信息”。
3. 父亲相关少玩梗，最多轻轻一刺，然后沉下来。
4. 装备文案可以搞笑，但效果必须诚实。
5. 传统武术文案不要全盘嘲讽，要强调“能压力测试的留下”。

示例：

```text
你摆出茂家拳起手式。
从远处看，确实有一代宗师的气势。
从近处看，像一个准备搬冰箱的人。
```

```text
你看见了对方的拳。
这是巨大的进步。
你没有躲开。
这是现实。
```

```text
父亲不是没教你绝招。
他是真的不会。
但他记得你第一次主动出门跑步的日子。
```

## 14. 风险与控制

### 风险 1：新增资源太多，HUD 变臃肿

控制：新增 `maw` 字段只在拳谱页和少数结算显示，HUD 不常驻展示。

### 风险 2：剧情战破坏玩家自由

控制：Day 8 必败，但让玩家操作一窗口，输法根据玩家准备变化。失败不是没收操作，而是叙事检验。

### 风险 3：最终战太难或太简单

控制：用目标制，不用纯 KO。玩家只要做出成长证据就能通过。

### 风险 4：文案太闹，父亲线失真

控制：父亲日记、Day 29 夜谈、结尾少用梗，保持温柔。

### 风险 5：`data.js` 与 `combat.js` 技能重复定义

控制：MVP 先同步修改两处；后续单独 Batch 做 `combatSkillFromData()` 的轻量统一，不在主线接入时顺手重构。

## 15. 最终交付标准

完成主线接入后，Demo 应该达到：

1. 玩家清楚知道自己是谁：茂家拳传人陆小闲。
2. 玩家体验到自己为什么会误判自己很强。
3. 玩家在 Day 8 被现实打醒。
4. 玩家在 Day 9 理解父亲不是骗子，而是笨拙地爱他。
5. 玩家在 Day 10-28 能自由训练，但所有成长都写回茂家拳谱。
6. 玩家在 Day 30 不需要成为天下第一，只要证明自己不是一阵风。
7. 结尾能落到：神功是假的，陪伴是真的；祖传是假的，不服输是真的；茂家拳从今天开始是真的。
