# Codex Context

## 项目简介

《了不起的武术模拟器》是一个 Phaser Web MVP：30 天城市武术成长模拟。玩家在城市里工作、训练、社交、处理事件、购买和使用物品，并通过阶段性实战检验成长。

## 核心循环

1. 选择当天目标：主线、待办、训练、赚钱、恢复或社交。
2. 在地点之间移动，承担时间、体力、现金和风险成本。
3. 通过训练、工作、事件和装备积累能力。
4. 进入战斗，用读板、选卡、短窗口自动交换和复盘形成战斗闭环。
5. 根据结算反馈调整下一天的路线。

## 视觉方向

- 都市武术题材，偏黑红金、P5 式斜切漫画 UI。
- Phaser 负责背景、角色 sprite、动画、伤害数字和 VFX。
- DOM 负责 HUD、底部导航、任务板、技能卡、装备、日志、弹窗和复杂说明文本。
- 当前部分 sprite strips 和 standee 是可替换运行底座，不是最终逐帧美术。

## 地点

核心地点包括出租屋、便利店、工地、公园、拳馆、武馆、MMA 馆、社区健身房、理疗馆和旧城区街道。城市大地图是按需打开的 overlay，不应默认占据所有地图页面空间。

## 底部导航

底部导航覆盖主要页签：

- 地图
- 人物
- 技能
- 背包
- 商店
- NPC/关系
- 日志
- 自检

移动端必须保证底部导航不遮挡核心按钮。

## 核心系统

- `GameStore` 负责状态、dispatch、存档、读档和 render model。
- `content/data.js` 存放地点、技能、敌人、物品、主线和行动数据。
- `simulation/state.js` 负责主流程、行动、出行、装备、训练小游戏、战斗入口和结算。
- `simulation/combat.js` 负责战斗解析、敌人回应和战斗公式。
- `dom/ui.js` 和 `dom/ui.css` 负责主要交互 UI。
- Phaser 场景负责加载资产和播放战斗表现。

## 代码原则

- 先理解现有数据结构和调用链，再做最小可验证改动。
- 优先数据驱动，避免在多个地方重复硬编码。
- 不默认改玩法数值、经济曲线、敌人属性、存档 key/version。
- 不默认扫描全仓库；先读 `FILE_MAP`，再选最小相关文件集。
- 每个非简单任务都要更新 `docs/TASK_PLAN.md` 并运行 `npm run build`。
