## 1. 核心设计结论

当前项目最强的方向不是继续做“30 天刷数值通关”，而是把 30 天压成一条清晰的成长戏剧：

> 陆小闲先靠误判和父亲的善意谎言撑起信念，再被真正的“一阵风”打醒，最后把拳击、身体、传统武术、MMA、防街头风险这些现实模块重新装进毛家拳里。

胜利定义不应是“毛家拳天下第一”，而是：

> 毛家拳以前可能只是零；到第 30 天，陆小闲把它练成了一个能站住的东西。

这能同时服务玩法、战斗、世界观和 UI：

- 玩法上：玩家不是刷一个总战力，而是在 30 天里修补具体短板。
- 数值上：早期“信念/误读”提供情绪推进，中期“真相/重铸”切换成长曲线，后期“目标战”验证模块组合。
- 战斗上：不追求一招秒人，核心体验是读懂对手、吃亏后修正、在窗口内完成目标。
- 叙事上：父亲不是骗子，而是用孩子能承受的方式托住他；世界观从荒诞转向有重量的现实武术。
- UI 上：界面应该从“可点按钮列表”升级成“30 天备战指挥台”。

## 2. 30 天结构重排

### 第一幕：信念与误读 Day 1-7

目标：让玩家相信陆小闲“好像真有点东西”，但所有胜利都要留下现实解释。

建议体验：

- Day 1：祖师爷点赞，建立荒诞口吻和父亲旧相册。
- Day 2：第一次小胜，胜因是对手更菜、轻敌或环境误判。
- Day 3：旧相册 / 父亲手写拳谱出现，解锁 `belief`。
- Day 5：公园冲突，玩家能打赢，但战斗后 UI 提示“这不是实战答案”。
- Day 7：朋友或小满点出“你赢的都不是会打的人”。

数值重点：

- `belief` 上升会提高士气、行动收益、失败后的恢复。
- `misread` 也随早期胜利上升，代表陆小闲把侥幸理解成门派实力。
- 早期不直接涨大量战斗硬数值，避免玩家误解为“神功有效”。

### 第二幕：一阵风 Day 8-9

目标：用一次可交互失败完成剧情转折。

Day 8 “一阵风”不做纯过场。玩家至少有一次可操作战斗窗口：

- 玩家可以防守、刺拳、后撤、观察。
- 系统展示对手读牌：重心稳、距离卡死、出手前没有多余动作。
- 无论结果如何，对手在第二窗口用“一阵风”结束战斗。
- 失败惩罚不应是重伤或永久损失，而是打掉 `misread`，打开 `truthRevealed`。

Day 9 父亲日记：

- 父亲承认毛家拳不是什么无敌拳。
- 父亲真正留下的是“别怕输、别乱欺负人、真要保护人就去学真东西”。
- UI 切换到“重铸阶段”。

数值重点：

- `misread` 清零或大幅下降。
- `belief` 不清零，而是转成 `fatherMemory`。
- 解锁 `reforge` 模块和主线目标清单。

### 第三幕：重铸 Day 10-28

目标：玩家不再证明祖传神功，而是把现实训练模块装进自己的打法。

重铸建议拆成 5 个模块：

| 模块 | 占比 | 对应地点 | 设计作用 |
| --- | ---: | --- | --- |
| 拳击 | 30% | boxing | 建立直拳、护架、距离、节奏 |
| 身体 | 20% | gym / worksite / physio | 支撑耐力、抗压、恢复 |
| 传统 | 20% | wuguan / park | 转化为平衡、推手、节奏破坏，不做玄学秒杀 |
| MMA | 20% | mma | 防抱摔、脱困、贴身风险 |
| 街头风险 | 10% | street / store | 识别多人、器械、情绪升级，强调止损 |

重铸阶段不建议让每个模块都变成独立复杂系统。最小闭环是：

- 每个模块 2-3 个关键训练动作。
- 每个模块有 1 个“验证战”或“事件验证”。
- 第 30 天目标战读取模块完成度，生成目标清单和战斗修正。

### 第四幕：上桌 Day 29-30

Day 29 烧烤夜：

- 不再发新系统。
- 回收父亲、小满、拳馆、工地、武馆、MMA 馆的人情。
- 根据玩家模块完成度给出不同台词。
- 明确第 30 天不是复仇，而是“真东西检验”。

Day 30 最终战：

最终战不要只看总战力。建议使用目标战结构：

- 撑过第一波压迫。
- 成功护住一次重拳。
- 用直拳打断一次推进。
- 被击中后完成一次恢复。
- 使用一次重铸技能。
- 保持冷静，不被热度/怒气带走。

胜负分层：

- 大胜：完成 5-6 个目标，结局是“毛家拳有了骨头”。
- 小胜：完成 3-4 个目标，结局是“还不够好，但已经不是假的”。
- 惜败：完成 2 个目标，结局是“你站住了，只是还没赢”。
- 崩盘：低于 2 个目标，结局不否定主线，只提示训练结构缺口。

## 3. 核心数值框架

### 3.1 资源层

当前已有资源 `money / fame / auth / heat / fatigue / morale / hp / sp / fitXp` 可以保留，但叙事语义需要收敛。

建议定位：

| 资源 | 新定位 | 主要影响 |
| --- | --- | --- |
| money | 生存压力 | 决定训练选择和恢复质量 |
| fatigue | 疲劳债 | 限制连续高强度训练，影响战斗失误率 |
| morale | 当天心气 | 影响训练收益下限、失败恢复 |
| auth | 门派信念/父亲记忆 | Day 9 前偏信念，Day 9 后偏父亲记忆 |
| heat | 冲突热度 | 街头/赛事风险，不应等同名气 |
| fame | 外界关注 | 解锁事件和压力，不直接等同强度 |
| fitXp | 身体底盘经验 | 支持解锁恢复和身体训练节点 |

### 3.2 战斗属性层

保留现有 `str/end/spd/rea/tec/tou/bal/jud`，但每个属性要有战斗职责。

| 属性 | 战斗职责 | 叙事解释 |
| --- | --- | --- |
| str | 打击威胁、架开能力 | 有劲但不是会打 |
| end | 长回合稳定性、疲劳衰减 | 工地和有氧给底盘 |
| spd | 出手和后撤窗口 | 年轻优势 |
| rea | 防守反应、读招修正 | 从挨打里学来的 |
| tec | 技术动作质量 | 拳馆/MMA 的硬成果 |
| tou | 挨打后不崩 | 不等于无脑硬扛 |
| bal | 被推、抱、撞后的稳定 | 传统/MMA 的交汇点 |
| jud | 读局面、选正确动作 | 主线后期最重要 |

### 3.3 重铸模块层

建议新增或抽象为可从现有 `styles` 读取的模块分数：

| 模块 | 来源 | 第 30 天作用 |
| --- | --- | --- |
| boxingCore | boxing + jab/straight/guard | 解锁“直拳打断推进” |
| bodyBase | fitXp + end/tou + recovery | 降低疲劳导致的失误 |
| traditionalBridge | traditional + bal/offbalance | 解锁“稳住重心/借力脱身” |
| grapplingDefense | mma + sprawl/escape | 防止被贴身拖死 |
| streetSense | street + talkdown/jud | 降低热度惩罚和情绪失控 |

不建议一开始新增复杂公式。第一版可用加权评分：

```text
reforgeScore =
  boxingCore * 0.30 +
  bodyBase * 0.20 +
  traditionalBridge * 0.20 +
  grapplingDefense * 0.20 +
  streetSense * 0.10
```

### 3.4 Day 8 与 Day 30 数值原则

Day 8：

- 不按普通战斗胜负结算。
- 目标是让玩家“有参与感地输”。
- 只结算少量经验、剧情状态和心理转折。

Day 30：

- 不只比较 `powerScore`。
- 用目标完成数决定结局层级。
- 允许玩家输了比赛但赢得主题，避免 30 天失败直接变成无意义。

## 4. 战斗重塑

### 4.1 现有战斗基础

当前 `combat.js` 已经有这些可利用基础：

- 敌人模板和技能模板。
- 行动预览 `previewPlayerAction`。
- 敌人读牌和反应 `chooseEnemyResponse`。
- 回合交换 `resolveCombatExchange`。
- 压力、疲劳、距离、架势等字段。
- UI 里已有 `readBoard` 展示基础。

因此第一阶段不需要推翻战斗系统。建议增强目标是：

> 让战斗从“点技能看伤害”变成“读对手意图，在正确窗口做正确动作”。

### 4.2 战斗动作分层

建议动作分 5 类：

| 类别 | 示例 | 正确用法 |
| --- | --- | --- |
| 探测 | jab / observe | 看反应、抢节奏，不追求伤害 |
| 进攻 | straight / combo | 在对手失衡或前压时打收益 |
| 防守 | guard / dodge | 抗压、换窗口 |
| 重置 | breath / footwork | 降压力、回耐力 |
| 转化 | sprawl / offbalance / escape | 应对抱摔、推搡、贴身 |

UI 上要让玩家知道“为什么现在该点这个”，不是只显示按钮。

### 4.3 敌人设计原则

每个主线敌人只保留一个清晰训练主题：

| 节点 | 对手 | 主题 | 玩家应学会 |
| --- | --- | --- | --- |
| Day 5 | 公园新手 | 乱冲 | 直拳/护架能处理低级冲突 |
| Day 8 | 一阵风 | 差距 | 真会打的人不吃玄学 |
| Day 12 | 武馆压力 | 重心 | 传统模块是平衡和节奏，不是神功 |
| Day 18 | 街头事件 | 风险 | 会打不等于该打，热度要控 |
| Day 24 | MMA 伙伴 | 贴身 | 防抱摔和脱困是必要短板 |
| Day 30 | 真东西 | 综合验证 | 你拼出的是自己的毛家拳 |

### 4.4 战斗数值护栏

- 普通训练战允许失败但给学习收益。
- 主线节点失败不应硬锁死进度，改为给不同后续目标。
- 街头战不鼓励刷收益，热度和恢复成本必须高于常规训练。
- 不做“传统武术大招秒杀现代格斗”的结果。
- 不做“现代训练完全否定父亲记忆”的结果。

## 5. 世界观叙事重排

### 5.1 世界观基调

建议基调是“市井现实 + 轻荒诞 + 真诚成长”。

不是纯燃漫，也不是冷嘲武术。陆小闲的世界应该具体：

- 家里有旧照片、旧护具、父亲的记账本。
- 工地和便利店提供生存压力。
- 公园和武馆承载传统人情。
- 拳馆和 MMA 馆提供现代训练秩序。
- 街头事件提醒玩家：现实不是擂台。

### 5.2 陆小闲人物弧线

| 阶段 | 他相信什么 | 玩家要感受到什么 |
| --- | --- | --- |
| Day 1-7 | 我家拳真有隐藏威力 | 好笑、轻飘、带点侥幸 |
| Day 8-9 | 原来我把很多事想错了 | 被打醒，但不羞辱玩家 |
| Day 10-20 | 我得补真正的短板 | 训练变得具体和有方向 |
| Day 21-28 | 我开始把东西接起来 | 多模块组合形成自己的打法 |
| Day 29-30 | 我不是替神话争面子 | 平静、站住、验证 |

### 5.3 父亲叙事原则

父亲不能写成“撒谎害孩子”。更稳的写法是：

- 他知道毛家拳不是无敌。
- 他也知道孩子需要一个能撑住童年的故事。
- 他真正希望陆小闲学会的是善意、分寸、抗压和保护人的能力。
- Day 9 的日记不是反转打脸，而是把童年故事翻译成成年人能理解的话。

## 6. UI 重塑方向

### 6.1 第一屏信息架构

当前 UI 不应继续像“很多地点按钮堆叠”。建议第一屏固定为 4 区：

1. 顶部：Day / 倒计时 / 疲劳 / 钱 / 当前主线状态。
2. 左侧或上方：今日目标与主线目标清单。
3. 中央：当前地点行动。
4. 右侧或下方：陆小闲状态、重铸模块、近期风险。

移动端则改为：

- 顶部固定状态条。
- 主线目标卡片优先。
- 行动列表按地点折叠。
- 战斗时只保留战斗关键信息，不显示次要经营信息。

### 6.2 重铸 UI

Day 9 后显示“重铸进度”，而不是只显示技能列表。

建议模块显示：

- 拳击核心：直拳、护架、距离。
- 身体底盘：耐力、恢复、抗压。
- 传统桥接：重心、推手、节奏。
- 贴身防守：防抱摔、脱困。
- 街头判断：控热度、识风险、会停手。

每个模块只显示：

- 当前等级 / 完成度。
- 下一条建议行动。
- 最接近的验证节点。

### 6.3 战斗 UI

战斗 UI 需要突出“读局面”：

- 敌人意图：前压、试探、抱摔、蓄重拳、退守。
- 我方风险：体力低、压力高、距离不对、架势破。
- 推荐不是直接告诉玩家答案，而是给一句可读提示：
  - “他现在重心压前，直拳比大摆拳更稳。”
  - “你被逼到近身了，先脱困再谈进攻。”
  - “热度已经高了，这场赢了也可能亏。”

### 6.4 视觉气质

UI 气质建议从“热闹面板”收敛成“训练日志 + 备战白板”：

- 字体层级更紧凑。
- 按钮不做夸张大卡片。
- 颜色不要一味热血红橙；用训练馆灰、纸张白、记号笔红、旧照片黄作为辅助。
- 主线事件可用父亲笔记样式，但不能影响可读性。

## 8. 不建议做的事

- 不建议马上重写战斗系统。
- 不建议新增大型依赖或切换 UI 框架。
- 不建议把传统武术写成玄学超能力。
- 不建议把 Day 8 做成纯播片。
- 不建议让 Day 30 只有赢/输两个结局。
- 不建议把所有模块都做成复杂小游戏。

## 9. 下一步推荐

下一轮建议从 Batch 19 开始：先补状态字段和迁移，不碰数值公式。这样可以让主线、UI、战斗目标都有统一数据入口，后面每一批都能小步验证。
